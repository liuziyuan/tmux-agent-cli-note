# 编辑器功能全方位设计文档

## 概述

设计两个编辑态（NORMAL 和 INSERT）的完整功能体系。NORMAL态保持Vim风格，INSERT态采用现代文本编辑器风格（支持鼠标、箭头键）。

## 第四部分：设计要点

### NORMAL 模式设计理念
- **保留Vim核心**：hjkl/dd/yy/p 等经典按键
- **简化复杂度**：不实现高级正则、宏、缩进等P2功能
- **核心完整性**：搜索/选择/删除/复制/粘贴是关键
- **渐进式增强**：P0→P1→P2 分阶段实现

### INSERT 模式设计理念
- **现代编辑器体验**：鼠标、标准快捷键
- **文本编辑优先**：删除、选择、快捷操作是核心
- **键盘和鼠标并重**：支持两种输入方式
- **易学易用**：Ctrl+C/V 等标准操作

### 两个模式的协作
1. **轻量编辑**：INSERT模式为主，快速输入和修改
2. **结构编辑**：NORMAL模式为辅，快速导航和重构
3. **模式切换**：Esc 随时返回NORMAL，i/A/o 快速进入INSERT

## 第五部分：实现路线图建议

### Phase 1（基础完善）- P0 优先级
**目标**：完成 P0 的所有待实现项
- NORMAL: 0/$点击/D删除到行末
- INSERT: Delete/Home/End/Ctrl+Home/Ctrl+End

### Phase 2（核心功能）- P1 高优先级
**目标**：实现 P1 中的高优先级项
- NORMAL: 搜索 + 替换 + 可视模式 + yy/p + u(撤销)
- INSERT: 鼠标支持 + Shift选择 + Ctrl+C/V/X/Z/Y

### Phase 3（完善体验）- P1 中优先级 + P2
**目标**：增强功能和高级特性
- NORMAL: w/b/J/宏/正则替换
- INSERT: Page Up/Down/自动补全/括号匹配

### 核心修改文件
- **src/editor.ts**：增加 handleKey 的分支逻辑

